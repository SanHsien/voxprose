# 聲成文 VoxProse 軟體品質管控流程 (QC Checklist)

本清單用於每次發布新版本前，確保核心功能在 Windows 上的穩定性。（本 repo 為 Windows 專用版）

## 1. 啟動與初始化 (Startup)
- [ ] **模型預載**: 啟動時是否正確顯示「載入中」指示器？
- [ ] **系統列圖示**: 托盤圖示是否正確顯示？右鍵選單是否能正常開啟？
- [ ] **Dashboard**: 點選「首頁/Dashboard」是否能正確彈出且不遮擋原本視窗（不再 AlwaysOnTop）？

## 2. 語音辨識與輸入 (STT & Input)
- [ ] **熱鍵測試 (PTT)**: 長按熱鍵（如右邊 Alt）錄音，放開後是否能正確辨識文字？
- [ ] **文字注入**: 文字是否能正確進入目前游標所在的輸入框？
- [ ] **空回傳處理**: 靜音錄音或噪音是否會導致崩潰？（應安靜消失或提示）
- [ ] **暫停/取消**: 錄音中按下 Esc 或其他取消操作，流程是否正確中斷且不注入？

## 3. AI 潤飾與翻譯 (LLM)
- [ ] **情境模式**: 切換「商務回應」、「情商大師」等情境，輸出語氣是否明顯改變？
- [ ] **快速翻譯**: 開啟翻譯（英/日）時，即使有其他情境設定，是否優先翻譯為目標語文？
- [ ] **禁令檢查**: AI 是否輸出「好的」、「這是您的結果」等廢話？（應嚴格禁止）
- [ ] **語系鎖定**: 非翻譯模式下，輸入英文是否仍回傳繁體中文（情境需要時）？

## 4. 系統指令與自動化 (Actions)
- [ ] **語音切換**: 說出「切換至國語模式」或「切換至商務英文」，系統設定是否即時連動變更？
- [ ] **內建工具**: 測試「天氣」、「時間」、「計算」等指令是否正確回饋。

## 5. 穩定性與邊界測試 (Stability)
- [ ] **重複切換模式**: 在錄音中、或連續快速點擊選單切換模式，是否不再發生 Access Violation 崩潰？
- [ ] **Dashboard 紀錄**: 錄音後檢查 Dashboard 的「分鐘數」與「字詞統計」是否正確累計。
- [ ] **長時間運行**: 背景待機 1 小時後，熱鍵功能是否依然靈敏？

## 6. 打包與部署 (Release)
- [ ] **Starter EXE**: `setup_win.bat` 是否成功編譯 `VoiceType4TW.exe`（啟動器檔名沿用既有打包鏈，未隨品牌改名）？桌面捷徑（顯示名稱「聲成文」）是否指向 EXE 且圖示正確？
- [ ] **乾淨安裝**: 刪除 `venv`/`.runtime` 後執行 `setup_win.bat` 全流程是否通過（Python 偵測、CUDA 條件安裝、模型下載）？
- [ ] **路徑檢查**: 資料是否正確寫入 `%APPDATA%\VoiceType4TW` 而非唯讀的安裝目錄？
